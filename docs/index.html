Tato dokumentace je zaměřena na všechny Cloud Functions funkce, tedy na funkce, které manipulují s daty v Cloud Firestore databázi na základě požadavků webové aplikace.

Rozdělení Projektu Simpleflashcards:

Projekt je rozdělen do dvou částí:

  • Cloud Functions - API, které bylo vytvořeno pro komunikaci mezi Cloud Firestore databází a klientem.
  • Samotná Simple Flashcards webová aplikace

Tato dokumentace je zaměřena na Cloud Functions. Dokumentaci pro webovou aplikaci můžete najít zde.

Databáze

Tento projekt využívá jako databázi službu Cloud Firestore. Jedná se o NoSQL databázi. Projekt využívá také službu Cloud Functions. Tato služba umožňuje vytvořit API a naprogramovat funce, díky kterým se dá do databáze přistupovat. V této dokumentaci je zaznamenána struktura databáze a popis jednotlivých funkcí.

Struktura databáze

Databáze je rozdělena do 3 částí:

  • collections - Dokumenty jednotlivých kolekcí.
  • decks - Dokumenty jednotlivých balíčků.
  • users - Dokumenty uživatelů a dokumenty se zaznamenanými pokroky u daných balíčků.

collections

V této části databáze jsou uloženy všechny kolekce. Každý dokument má název podle ID dané kolekce. Tyto ID jsou automaticky generovány funkcemi z Cloud Firestore. V dokumentech jsou uloženy následující informace:

  • [string] colName - Název kolekce, který uživatel zadal.
  • [string] colDescription - Popis dané kolekce, který uživatel vytvořil.
  • [string] creatorId - ID uživatele, který kolekci vytvořil.
  • [Array<string>] deckArray - Pole, které obsahuje seznam ID balíčků, které jsou obsaženy v dané kolekci.

decks

V této části databáze jsou uloženy všechny balíčky. Každý dokument má název podle ID daného balíčku. Tyto ID jsou automaticky generovány funkcemi z Cloud Firestore. V dokumentech jsou uloženy následující informace:

  • [string] deckName - Název balíčku, který uživatel zadal.
  • [string] deckDescription - Popis daného balíčku, který uživatel vytvořil.
  • [string] deckImage - Adresa obrázku daného balíčku. Pokud uživatel žádný balíček nenahrál, je zde zapsána adresa pro výchozí obrázek.
  • [string] creatorId - ID uživatele, který balíček vytvořil.
  • [boolean] private - Informace, která udává, jestli je balíček veřejný nebo soukromý.
  • [Array<Object>] cardArray - Pole, které obsahuje objekty jednotlivých karet. Ty obsahují následující informace:
    • [string] cardId - ID dané kartičky, které bylo vygenerováno funkcí newId.
    • [string] body1 - Text přední strany kartičky.
    • [string] body2 - Text zadní strany kartičky.

users

V této části jsou uloženy všechny informace o uživatelích a jejich pokroku v učení. Každý dokument má název podle ID daného uživatele. ID jsou automaticky generovány službou Firebase Authentication při registraci uživatele. V jednotlivých dokumentech této části databáze jsou pak uloženy následující informace:

  • [string] username - Uživatelské jméno.
  • [string] bio - Popis uživatelského profilu.
  • [string] email - Emailová adresa uživatele.
  • [Array<string>] createdDecks - Pole, které obsahuje ID všech balíčků, které uživatel vytvořil.
  • [Array<string>] pinnedDecks - Pole, které obsahuje ID všech balíčků, které uživatel připnul.
  • [Array<string>] createdCollections - Pole, které obsahuje ID všech kolekcí, které uživatel vytvořil.
  • [Array<string>] pinnedCollections - Pole, které obsahuje ID všech kolekcí, které uživatel připnul.

deckProgress

Ke každému dokumentu v této části databáze je přiřazena skupina dokumentů, která se nazývá deckProgress. Tato skupina obsahuje dokumenty, které mají název podle ID balíčků, které se uživatel učil. V každém tomto dokumentu je pak seznam kartiček, které se uživatel učil společně s zaznamenaným pokrokem u každé karty. V každém dokumentu jsou uloženy následující informace.

  • [string] deckId - ID daného balíčku (Je identické s názvem dokumentu. V dokumentu je zapsáno, aby bylo možné dokumenty hromadně prohledávat v databázi.).
  • [Array<Object>] cardArray - Pole, které obsahuje objekty karet, které se uživatel učil. Ty obsahují následující informace:
    • [string] cardId - ID dané kartičky.
    • [number] understandingLevel - Číslo, které udává, jak dobře uživatel rozumí dané kartičce.

Uživatelské účty

Registrace

Když se uživatel zaregistruje do Aplikace, je zaregistrován do služby Firebase Authentication. Zde jsou pak uloženy např. přihlašovací údaje uživatele (email a heslo) nebo ID uživatele. Zároveň je při registraci vytvořen dokument v databázi v části users. V tomto dokumentu se pak ukládají ostatní informace o uživateli (viz výše).

Přihlašování a odhlašování

Přihlášení uživatel probíhá takto: Klient zazádá o přihlášení. Cloud funkce získá ze služby Firebase Authentication idToken uživatele a ten pak odešle zpět klientovi. Tam se pak idToken uloží do localStorage a je přidán do headeru dalších požadavků na server.

Při odhlášení uživatele je idToken odstraněn z localStorage klienta a z headeru požadavků.
