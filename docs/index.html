Home

Tato dokumentace je zaměřena na Cloud Functions, tedy na funkce, které tvoří API projektu a manipulují s daty v Cloud Firestore databázi na základě požadavků webové aplikace. Funkce běží na serverech služby Firebase.

Rozdělení Projektu Simpleflashcards:

Projekt je rozdělen do dvou částí:

  • Cloud Functions - API, které bylo vytvořeno pro komunikaci mezi Cloud Firestore databází a klientem.
  • Samotná Simple Flashcards webová aplikace

Tato dokumentace je zaměřena na Cloud Functions. Dokumentaci pro webovou aplikaci můžete najít zde.

Struktura dokumentace:

Funkce

  • collection - Funkce, které se starají o vytváření, upravování, připínání a odepínání kolekcí a získávání dat kolekcí.
  • collectionCards - Funkce, které získávají z databáze karty na učení z dané kolekce a ukládají pokrok uživatele.
  • deck - Funkce, které se starají o vytváření, upravování, připínání a odepínání balíčků a získávání dat balíčků.
  • deckCards - Funkce, které získávají z databáze karty na učení z daného balíčku a ukládají pokrok uživatele.
  • functions - Funkce používané dalšími moduly.
  • user - Funkce spravující data uživatelů.

Ostatní

  • index - Javascript modul, v němž se z funkcí vytvoří API.

util

  • admin - Javascript modul pro spuštění Firebase Admin SDK.
  • authMiddleware - Funkce, která slouží k ověření uživatele.
  • other - Konstanty používané ve funkcích.

Databáze

Tento projekt využívá jako databázi službu Cloud Firestore. Jedná se o NoSQL databázi. Projekt využívá také službu Cloud Functions. Tato služba umožňuje vytvořit API a naprogramovat funkce, díky kterým se dá do databáze přistupovat. V této dokumentaci je zaznamenána struktura databáze a popis jednotlivých funkcí.

Struktura databáze

Databáze je rozdělena do 3 částí:

  • collections - Dokumenty jednotlivých kolekcí.
  • decks - Dokumenty jednotlivých balíčků.
  • users - Dokumenty uživatelů a dokumenty se zaznamenanými pokroky u daných balíčků.

collections

V této části databáze jsou uloženy všechny kolekce. Každý dokument obsahuje data jedné kolekce a má název podle ID dané kolekce. Tyto ID jsou automaticky generovány funkcemi z Cloud Firestore. V dokumentech jsou uloženy následující informace:

  • [string] colName - Název kolekce.
  • [string] colDescription - Popis kolekce, který uživatel zadal. Pokud uživatel nezadal popis kolekce, má hodnotu null.
  • [string] creatorId - ID uživatele, který kolekci vytvořil.
  • [boolean] private - Informace, která udává, jestli je kolekce veřejná nebo soukromá.
  • [Array<string>] deckArray - Pole, které obsahuje ID balíčků, které jsou obsaženy v dané kolekci.

decks

V této části databáze jsou uloženy všechny balíčky. Každý dokument obsahuje data jednoho balíčku a má název podle ID daného balíčku. Tyto ID jsou automaticky generovány funkcemi z Cloud Firestore. V dokumentech jsou uloženy následující informace:

  • [string] deckName - Název balíčku.
  • [string] deckDescription - Popis daného balíčku, který uživatel zadal. Pokud uživatel nezadal popis balíčku, má hodnotu null.
  • [string] deckImage - Adresa obrázku daného balíčku. Pokud uživatel žádný balíček nenahrál, je zde zapsána adresa pro výchozí obrázek.
  • [string] creatorId - ID uživatele, který balíček vytvořil.
  • [boolean] private - Informace, která udává, jestli je balíček veřejný nebo soukromý.
  • [Array<Object>] cardArray - Pole, které obsahuje objekty jednotlivých karet balíčku. Každý objekt obsahuje následující informace:
    • [string] cardId - ID dané kartičky, které bylo vygenerováno funkcí newId.
    • [string] body1 - Text přední strany kartičky.
    • [string] body2 - Text zadní strany kartičky.

users

V této části jsou uloženy všechny informace o uživatelích a jejich pokroku v učení. Každý dokument obsahuje data jednoho uživatele a má název podle ID daného uživatele. ID jsou automaticky generovány službou Firebase Authentication při registraci uživatele. V jednotlivých dokumentech této části databáze jsou pak uloženy následující informace:

  • [string] username - Uživatelské jméno.
  • [string] bio - Popis uživatelského profilu.
  • [string] email - Emailová adresa uživatele.
  • [Array<string>] createdDecks - Pole, které obsahuje ID všech balíčků, které uživatel vytvořil.
  • [Array<string>] pinnedDecks - Pole, které obsahuje ID všech balíčků, které uživatel připnul.
  • [Array<string>] createdCollections - Pole, které obsahuje ID všech kolekcí, které uživatel vytvořil.
  • [Array<string>] pinnedCollections - Pole, které obsahuje ID všech kolekcí, které uživatel připnul.

deckProgress

Ke každému souboru v této části databáze (users) je přiřazena skupina dokumentů, která se nazývá deckProgress. Dokumenty těchto skupin jsou určeny k uchovávání pokroku uživatele u balíčků a mají název podle ID daného balíčku. V každém dokumentu je pak uložen seznam karet balíčku, které se uživatel učil, společně se zaznamenaným pokrokem u každé karty. V každém dokumentu jsou tedy uloženy následující informace:

  • [string] deckId - ID daného balíčku (Je identické s názvem dokumentu. V dokumentu je zapsáno, aby bylo možné dokumenty hromadně prohledávat v databázi.).
  • [Array<Object>] cardArray - Pole, které obsahuje objekty karet balíčku, které se uživatel učil. Ty obsahují následující informace:
    • [string] cardId - ID dané kartičky.
    • [number] understandingLevel - Číslo, které udává, jak dobře uživatel rozumí dané kartičce.

Uživatelské účty

Registrace

Když se uživatel zaregistruje do aplikace, je zaregistrován do služby Firebase Authentication. Zde jsou pak uloženy např. přihlašovací údaje uživatele (email a heslo) nebo ID uživatele. Zároveň je při registraci vytvořen dokument v databázi v části users. V tomto dokumentu se pak ukládají ostatní informace o uživateli (viz výše).

Přihlašování a odhlašování

Přihlášení uživatele probíhá takto: Klient zažádá o přihlášení. Cloud funkce získá ze služby Firebase Authentication idToken uživatele a ten pak odešle zpět klientovi. Tam se pak idToken uloží do localStorage a je přidán do headeru dalších požadavků na server. Při zpracovávání těchto požadavků se pak vždy ověří platnost daného idTokenu. To zajišťuje funkce authMiddleware.

Při odhlášení uživatele je idToken odstraněn z localStorage klienta a z headeru požadavků.

Instalace a úprava funkcí

Pro užití funkcí pro vlastní projekt je třeba vytvořit a nastavit nový Firebase projekt (více zde).

Po vytvoření projektu příkazem firebase init functions stačí do adresáře projektu přidat složku functions a v této složce pak spustit příkaz npm install. Po případné úpravě kódu daných funkcí je třeba soubory nahrát na server příkazem firebase deploy.
